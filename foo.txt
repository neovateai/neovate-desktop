⏺ Now I have a complete picture. Here's my research summary:

  ---
  调研结果

  当前架构

  Session 列表通过 acpx 库获取:
  - listSessionsForAgent(agentCommand) 读取 ~/.acpx/sessions/*.json
  - 按 agentCommand ("npx -y @zed-industries/claude-agent-acp") 过滤
  - 返回 SessionRecord（含 title, cwd, timestamps）

  Claude Code CLI Session 存储格式

  路径: ~/.claude/projects/<encoded-project-path>/<session-uuid>.jsonl

  路径编码规则: / → -，例如:
  ~/.claude/projects/-Users-chencheng-Documents-Code-github-com-neovateai-neovat
  e-desktop/

  JSONL 每行一个 JSON 对象，类型包括:
  - user — 用户消息（含 message.role, message.content, cwd, sessionId,
  timestamp）
  - assistant — 助手回复（含 message.content（包括 thinking blocks）,
  message.model, message.usage）
  - system — 系统事件（slash commands 等）
  - file-history-snapshot — 文件历史快照
  - progress / queue-operation — 内部状态

  关键差异: Claude Code CLI 没有索引文件或元数据文件，所有信息内联在 JSONL
  中。没有独立的 title 字段——需要从第一条用户消息提取。

  三个可行方案

  方案 A: 直接文件系统扫描（推荐）

  在 main 进程新增一个独立模块，直接扫描 ~/.claude/projects/ 目录:

  1. 列表: 根据当前 cwd 计算编码路径，扫描 *.jsonl 文件，读取前几行提取
  sessionId、timestamp、第一条用户消息作为 title
  2. 加载: 解析完整 JSONL，将 user/assistant entries 转换为现有 AcpMessage 格式
  3. 显示: 在 SessionList 中以独立分区或标记展示（如 "CLI Sessions"）

  优点:
  - 不依赖 Claude Code CLI 进程
  - 读取速度快（只需解析文件前几行即可获取列表元数据）
  - 可以做成只读模式，不需要 ACP 连接

  缺点:
  - 需要维护 JSONL 格式解析器，Claude Code 更新格式可能 break
  - 只读——无法在这些 session 上继续对话

  方案 B: 通过 Claude Code CLI --resume + --print

  利用 CLI 自身能力:
  claude --resume <sessionId> --print --output-format stream-json

  优点:
  - 不需要自己解析 JSONL 格式
  - 如果格式变了，CLI 自己处理

  缺点:
  - --resume 实际上会恢复会话并开始新一轮交互，不是纯读取历史
  - 需要启动 Claude Code CLI 进程，开销大
  - 列表发现仍需要自己扫描文件系统

  方案 C: 将 Claude Code CLI 也注册为 ACP Agent

  通过 acpx 的 agent 注册机制，让 Claude Code CLI 作为另一个 agent 接入:

  优点:
  - 复用现有的整套 session 管理架构
  - 可以继续对话

  缺点:
  - Claude Code CLI 本身不是 ACP agent，需要一个 adapter/wrapper
  - @zed-industries/claude-agent-acp 已经是这个 wrapper，但它创建的 session
  存储在 ~/.acpx/sessions/，不共享 CLI 的 ~/.claude/projects/

  我的建议

  方案 A 最实际。核心工作量:

  1. 新增 claude-cli-sessions.ts — 扫描 ~/.claude/projects/ 目录，解析 JSONL
  提取 SessionInfo
  2. 修改 router.ts 的 listSessions — 合并 acpx sessions 和 CLI sessions，加个
  source: "acpx" | "claude-cli" 字段
  3. 修改 SessionList UI — 展示 source 标记，CLI sessions 标记为只读
  4. 新增 loadCliSession — 解析完整 JSONL 转换为 StreamEvent[] 用于渲染

  要不要我进一步细化某个方案，或者直接开始实现？
